# RISEN CARE Design System

Version 1.0

---

# Purpose

The RISEN CARE Design System provides a consistent foundation for designing products that help care professionals better understand residents.

It defines not only visual styles, but also the principles for presenting information, designing interactions, and creating a coherent user experience.

Every component should support the Design Philosophy.

> Deepen understanding.

---

# Design Goals

The RISEN CARE interface should feel:

* Calm
* Clear
* Trustworthy
* Human-centered
* Explainable

Users should never feel overwhelmed.

Users should always know where to look next.

---

# Visual Principles

## Simplicity

Remove unnecessary elements.

Every component must have a purpose.

---

## Clarity

Information hierarchy is more important than decoration.

Important information appears first.

Supporting information appears later.

---

## Consistency

The same information should always look the same.

Patterns create confidence.

---

## Progressive Disclosure

Show only what users need now.

Reveal more information only when requested.

---

## Explainability

Every AI-generated insight should have accessible supporting evidence.

Nothing should feel like a black box.

# UX Writing

Use familiar language.

Avoid technical terminology.

Prefer:

利用者理解

instead of

Knowledge Graph

Prefer:

判断の根拠

instead of

Explainability

Always write for care professionals.

---

# Accessibility

High contrast.

Readable typography.

Large touch targets.

Keyboard accessible.

Color is never the only indicator.

---

# Figma Rules

Use Auto Layout.

Use Components.

Use Variables.

Use Design Tokens.

Avoid duplicate components.

Everything should be reusable.

---

# Design Checklist

Before creating a screen, ask:

Does this improve understanding?

Is this the simplest possible design?

Can users immediately find the most important information?

Can users reach the supporting evidence?

Can this component be reused?

If the answer is "No", redesign it.
